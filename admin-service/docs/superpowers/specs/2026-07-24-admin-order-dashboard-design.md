# 관리자 주문 대시보드 조회 설계

## 목적

관리자가 주문 현황을 페이지 단위로 조회할 수 있도록 주문 요약과 주문 상품별 내역을 함께 제공한다. 이 기능은 환불을 직접 처리하지 않으며, 환불된 주문과 상품의 상태를 대시보드에 표시하는 읽기 전용 기능이다.

## 응답 구조

각 주문 항목은 다음 정보를 반환한다.

- 주문번호
- 구매자: 사용자 ID, 이름, 프로필 사진 URL
- 총 주문 금액
- 전체 주문 상태
- 주문일
- 주문 상품 목록

각 주문 상품 항목은 다음 정보를 반환한다.

- 판매자: 사용자 ID, 이름, 프로필 사진 URL
- 상품명 스냅샷
- 상품 금액 스냅샷
- 주문 상품 상태

주문 상태는 주문 전체의 상태이고, 주문 상품 상태는 개별 상품의 처리 상태다. 따라서 부분 환불 주문도 상품별 상태를 통해 정확히 표시할 수 있다.

## 조회 흐름

1. 상태 조건과 페이지 조건으로 주문 ID를 생성일 내림차순으로 조회한다.
2. 해당 페이지의 주문 ID에 속한 모든 주문 상품을 한 번에 조회한다.
3. 주문의 구매자 ID와 주문 상품의 판매자 ID를 중복 없이 수집한다.
4. 수집한 사용자 ID의 이름과 프로필 사진 URL을 한 번에 조회한다.
5. 주문별로 주문 상품을 묶고, 사용자 정보를 조립하여 페이지 응답을 만든다.

이 순서는 주문과 주문 상품의 조인으로 페이지 크기가 왜곡되는 것을 막고, 주문·상품·사용자별 반복 조회를 방지한다.

## 데이터 소유권과 누락 처리

- `admin-service`는 `order-service`와 `user-service`가 소유한 테이블을 읽기 전용으로 재매핑한다. 주문 상태·주문 상품 상태·환불 정보는 이 서비스에서 변경하지 않는다.
- 주문 번호와 구매자 ID는 주문 미러 엔티티에, 주문 상품 상태는 주문 상품 미러 엔티티에 추가로 매핑한다.
- 사용자 미러 엔티티에는 프로필 사진 URL을 추가로 매핑한다.
- 사용자 정보가 존재하지 않으면 이름은 `알 수 없음`, 프로필 사진 URL은 `null`로 반환한다.

## 오류 처리

- 지원하지 않는 주문 상태 필터는 기존과 같이 400 응답으로 처리한다.
- 페이지에 주문이 없으면 빈 목록과 정상 페이지 메타데이터를 반환하며 사용자 조회를 수행하지 않는다.
- 사용자 정보가 일부 누락되어도 주문 조회 전체를 실패시키지 않는다.

## 테스트 범위

- 주문 페이지네이션과 상태 필터
- 한 주문에 속한 모든 주문 상품의 반환
- 구매자와 판매자 이름·프로필 사진의 일괄 매핑
- 상품 금액·상품 상태·주문번호·구매자 매핑
- 사용자 정보가 없을 때의 기본값
- 빈 주문 페이지에서 사용자 조회 생략

## 범위 제외

- 관리자 환불 실행 API
- 주문 또는 주문 상품 상태 변경
- 결제 취소·정산 재계산·환불 사유 관리
