# commit

staged 변경사항을 분석해 Git 컨벤션에 맞는 커밋 메시지를 제안하고 승인 후 커밋한다.

## 핵심 원칙

- 커밋 메시지 형식·타입 목록은 `@.claude/rules/git-convention.md`를 따른다
- **승인 전에는 커밋하지 않는다**
- 타입은 diff를 직접 읽고 판단한다. 사용자가 별도로 말하지 않아도 된다

---

## 실행 순서

### 1. 사전 점검

```bash
git status --porcelain
```

결과를 보고 세 가지 경우로 분기한다.

**staged도 unstaged도 없음**:
```
커밋할 변경사항이 없습니다.
```
종료.

**staged 없고 unstaged만 있음**:
변경 파일 목록을 보여주고 선택지를 제시한다.
```
staged 변경사항이 없습니다. 아래 파일이 수정되어 있습니다:
  M src/main/java/.../UserController.java
  M src/main/java/.../UserService.java

a) 전체 stage 후 계속 (git add -A)
b) 직접 git add 후 /commit 재실행
c) 취소
```
- a 선택 시: `git add -A` 실행 → 2단계로 진행
- b 선택 시: 안내 후 종료
- c 선택 시: 종료

**staged가 있음**: 바로 2단계로 진행.

---

### 2. diff 분석

```bash
git diff --staged --stat
git diff --staged
```

두 결과를 모두 읽고 타입을 결정한다.

**타입 판단 기준**:

| 타입 | 해당하는 변경 |
|------|--------------|
| `feat` | 새 파일·새 기능·새 API 엔드포인트 추가 |
| `fix` | 버그 수정, 잘못된 로직 교정 |
| `refactor` | 기능 변화 없이 구조·네이밍·추상화 개선 |
| `docs` | .md 파일, Javadoc, 주석만 변경 |
| `chore` | build 설정, 의존성, .gitignore 등 비기능 파일 |
| `style` | 들여쓰기·공백·포맷만 변경 (로직 무변경) |
| `test` | 테스트 코드만 추가·수정 |

복합 변경이면 **주된 변경**의 타입을 선택한다. 예: 기능을 추가하면서 테스트도 같이 작성했다면 `feat`.

---

### 3. 메시지 제안

분석 결과를 바탕으로 메시지를 제안한다. 내용은 한국어로 작성하되 50자 이하를 권장한다.

메시지가 50자를 넘으면 제안 텍스트 앞에 `(메시지가 깁니다. 줄여보세요) ` 를 붙인다.

---

### 4. 사용자 승인

`AskUserQuestion` 도구를 사용해 아래 형식으로 묻는다.

- question: `"커밋 메시지를 확인해주세요: \`{제안 메시지}\`"`
- header: `"커밋 승인"`
- options:
  - label: `"승인"`, description: `"제안 메시지 그대로 커밋"`
  - label: `"메시지 수정"`, description: `"직접 입력한 메시지로 커밋"`
  - label: `"취소"`, description: `"커밋하지 않고 종료"`

**승인** 선택 시: 제안 메시지로 바로 커밋  
**메시지 수정** 선택 시: "새 커밋 메시지를 입력해주세요:" 라고 텍스트로 요청 후, 입력받은 메시지가 `<타입>: <내용>` 형식에 맞지 않으면(타입이 허용 목록 밖이거나 콜론 없음) 경고 후 재입력 요청  
**취소** 선택 시: staged 상태 유지, 커밋하지 않고 종료

---

### 5. 커밋 실행

```bash
git commit -m "<메시지>"
```

성공 시:
```
커밋 완료: abc1234
feat: 사용자 프로필 조회 API 구현
```

실패 시(pre-commit hook 등): 에러 원문을 그대로 출력하고 안내한다.
```
커밋 실패: 위 오류를 수정한 뒤 다시 /commit을 실행하세요.
```

---

## 엣지 케이스

| 상황 | 처리 |
|------|------|
| staged 없음, unstaged 있음 | 파일 목록 + 스테이징 선택지 제시 |
| staged도 unstaged도 없음 | "커밋할 변경사항이 없습니다." 종료 |
| 복합 타입 변경 | 주 변경 타입 선택, 제안 시 근거 한 줄 표시 |
| 메시지 50자 초과 | 제안 아래 줄이기 권고 (강제하지 않음) |
| 허용 타입 외 메시지 입력 | 경고 후 허용 타입 목록 안내, 재입력 요청 |
| pre-commit hook 실패 | 에러 원문 출력 + "위 오류 수정 후 재실행" |
